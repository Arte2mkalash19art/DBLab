## 1. Вербальна модель предметної області
Система призначена для автоматизації процесу продажу авіаквитків. Вона зберігає дані про країни, міста, типи літаків та конкретні рейси. Користувачі можуть купувати квитки на рейси, при цьому система автоматично розраховує фінальну вартість з урахуванням класу місця (Economy, Business, First) та додаткових зборів (паливний збір, страховка тощо).

## 2. Бізнес-правила
1. **Унікальність:** Кожен рейс (`FLIGHT`) повинен мати унікальний номер.
2. **Географія:** Місто вильоту не може збігатися з містом прибуття.
3. **Вікові обмеження:** Ціна квитка та збори не можуть бути від’ємними.
4. **Цілісність:** Видалення країни неможливе, якщо в ній зареєстровані міста (FK constraint).

# Лабораторна робота №2: Створення запитів у MS SQL

**Виконав:** Назарійчук Артем Богданович  
**Предметна область:** Система бронювання авіарейсів (MyFlightBase)

## Опис проекту
Цей репозиторій містить скрипти для розгортання та тестування бази даних авіарейсів. Проект включає структуру таблиць, автоматичне наповнення даними та аналітичні запити.

## Структура файлів
* **SETUP.SQL** — створення бази даних та таблиць (`COUNTRY`, `CITY`, `PLANE_TYPE`, `FLIGHT`).
* **INSERT.SQL** — скрипт для автоматичного наповнення кожної таблиці (по 100 записів) за допомогою циклів `WHILE`.
* **UPDATE.SQL** — приклади модифікації та видалення даних.
* **QUERY.SQL** — набір запитів для аналізу даних (фільтрація, об'єднання JOIN, агрегація).

## Як запустити
1. Підключіться до локального сервера MS SQL Server (наприклад, `.\SQLEXPRESS`).
2. Виконайте файл `SETUP.SQL` для створення структури.
3. Виконайте `INSERT.SQL` для наповнення бази.
4. Використовуйте `QUERY.SQL` для перевірки працездатності запитів.
